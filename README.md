# TestTask42
Для решния этой задачи был использован контейнер list, так как он реализован с помощью двусвязых списков, что требуется по условию. <br />
Рассмотрим алгоритм решения. В функции main с клавиатуры вводится число, обозначающее клиество элементов для добавления в обрабатываемый список. <br />
Далее, заполняется список случайными восьмибитовыми значениями, после этого, полученный список копируется. <br />
Далее, запускаются два потока, которые будут выполнять функцию, подсчитывающую количество нулевых или единичных битов в каждом элементе списка. Отметим, что различие между подсчётом нулей или единиц выбирается с помощью аргумента типа bool, с помощью этого же аргумента выбирается, направление обхода списка потоком. <br />
Рассмотрим подробнее функцию ListChecker. Эта функция выполняет все действия с копией исходного списка. 
В начале, происходит выбор направления обхода списка и элементов для поиска. После этого, запускается цикл, условием выхода из которого является отсутсвие элементов в списке. <br />
Далее, блокируется мьютекс, выполняется проверка на наличие элементов, после чего происходит учёт элемента (т.е. подсчёт количества нулей или единиц), затем учтённый элемент удаляется из копии списка  мьютекс разблокируется. <br />
Затем, происходит увеличение счётчика пройденных потоком элементов.<br />
Таким образом, выполняются все условия поставленного задания. <br />
Рассмотрим пример работы: <br />

<br /> Было введено число 1, т.е. нужно сгенерировать одно восьмибитное число. Получилось число 19. <br />
В двоичном виде 19 можно представить как: 10011. Но так как число восмибитное, то правильно будет дописать незначащие нули в начале, т.е. получим 00010011 <br />
Всего получили пять нулей, что сответствует выводу программы. <br />
